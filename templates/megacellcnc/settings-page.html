Celery Beat – Geräte-Status

Hintergrundaufgabe check_all_devices prüft regelmässig, ob deine Testgeräte erreichbar sind (wie in den Celery-Logs). Das Intervall gilt für den django-celery-beat Eintrag in der Datenbank (Scheduler: DatabaseScheduler).

Bereich 5–3600 Sekunden. Kleinere Werte = häufigere Logs und Netzlast.

Änderungen werden vom Beat-Prozess meist innerhalb weniger Sekunden übernommen. Falls nichts passiert: docker-compose restart celerybeat (oder den Celery-Beat-Container neu starten).

Database Backup & Restore
Backup erstellen

Erstelle ein vollständiges Backup der Datenbank. Das Backup wird als komprimierte JSON-Datei (.json.gz) heruntergeladen.

Backup wiederherstellen

Wähle dein Betriebssystem für die passende Anleitung:

  • Linux / macOS
  • Windows

Terminal öffnen und im Projektverzeichnis ausführen:

# Backup entpacken
gunzip megacnc_backup_XXXXXX.json.gz

# Backup wiederherstellen
docker-compose exec web python manage.py loaddata \
  megacnc_backup_XXXXXX.json

# Oder Datei ins Container kopieren
docker cp megacnc_backup_XXXXXX.json megacnc-web-1:/app/
docker exec megacnc-web-1 python manage.py loaddata \
  megacnc_backup_XXXXXX.json

PowerShell öffnen:

# Mit 7-Zip entpacken
7z e megacnc_backup_XXXXXX.json.gz

# Datei in Container kopieren
docker cp megacnc_backup_XXXXXX.json megacnc-web-1:/app/

# Backup wiederherstellen
docker exec megacnc-web-1 python manage.py loaddata `
  megacnc_backup_XXXXXX.json

# Datei wieder löschen
docker exec megacnc-web-1 rm megacnc_backup_XXXXXX.json
Achtung: Ein Restore überschreibt alle aktuellen Daten!
